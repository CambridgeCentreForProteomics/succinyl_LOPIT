Finally, lets look at the enriched GO terms for the succinylated nuclear proteins. As part of this, we will consider the potential biasing factor of protein abundance. Note that the bias (if present) would be technical (easier to detect high abundance), even if more abundant proteins are more likely to have a least one succinylated copy, since the proportion succinylated should not be dependent on abundance, so it still comes down to a difference in the ability to detect low vs high frequency events. We will use PaxDB for this, specifically the Geiger et al U-2 OS data.

OK, so goseq::nullp doesnt fit the relationship quite right. Below, we allow for more knots to improve the fit

my_nullp <- function (DEgenes, genome, id, bias.data = NULL, plot.fit = TRUE,
                      nknots=6) # default value in goseq::nullp
{
    if (!is.null(bias.data) & length(bias.data) != length(DEgenes)) {
        stop("bias.data vector must have the same length as DEgenes vector!")
    }
    bias.data = unfactor(bias.data)
    DEgenes = unfactor(DEgenes)
    if (is.null(bias.data)) {
        bias.data = getlength(names(DEgenes), genome, id)
    }
    pwf = rep(NA, length(DEgenes))
    w = !is.na(bias.data)
    pwf[w] = goseq::makespline(bias.data[w], DEgenes[w], nKnots = nknots)
    out = data.frame(DEgenes = DEgenes, bias.data = bias.data, 
        pwf = pwf, stringsAsFactors = FALSE)
    rownames(out) = names(DEgenes)
    if (plot.fit) {
        plotPWF(out, ylim=c(0, 0.6))
    }
    out
}

pwf_more_knots <- my_nullp(is_suc, bias.data=bias, nknots=19)
